General steps:-
--------------
1. Convert each interval/range into events.
   - Start event (+)
   - End event (-)

2. Order the events.
   - Sort by coordinate/time, this is O(nlog n)
   - OR use a difference array if coordinates are small and bounded to avoid sorting, this is O(n)

3. Sweep from left to right.
   - Maintain the current active state.
   - Update the answer at each event.





Sweep Line
    │
    ├── Small, dense coordinate range
    │      ↓
    │   Difference Array
    │
    ├── Large/sparse coordinates + offline problem
    │      (all events known upfront)
    │      ↓
    │   Event List + Sort
    │
    └── Large/sparse coordinates + online problem
           (events arrive incrementally)
           ↓
        TreeMap (ordered map)


Reasoning in simple words (my own wordings):-
--------------------------------------------
1. if the problem is offline, i.e., intervals/segments are given before hand, then TreeMap approach is ruled out
2. if Coordinate range is bounded, then go with diff. array, no sorting needed
3. but if range is sparse/too large (10^9 or so) go with event list and sorting approach
