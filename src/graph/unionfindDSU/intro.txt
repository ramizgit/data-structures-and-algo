Intro:-
Union-Find (DSU) is a data structure used to maintain groups of connected nodes.
All nodes in the same group ultimately share the same root parent,
which makes it very efficient to check whether two nodes belong to the same group and to merge two groups together.

Union-Find is primarily used on "undirected graphs" because connectivity is symmetric:

1. Find(x)
👉 Returns:
“Which group does x belong to?”

2. Union(x, y)
👉 Merges:
“Put x and y into the same group”

Key Difference (Golden Line)
DFS/BFS explores the graph, while Union-Find maintains connectivity information.


Pattern Recognition (VERY IMPORTANT)
Use Union-Find when you see:
👉 “Are these two connected?”
👉 “Merge groups dynamically”
👉 “Detect cycle while adding edges”
👉 “Groups keep changing”


Do these in order:
Redundant Connection
Number of Connected Components
Accounts Merge
💥 This gives you ~80% coverage of UF interview questions
